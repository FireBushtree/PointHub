# 前后端接口契约

## 1. 调用链路

- 前端统一通过 `@tauri-apps/api/core` 的 `invoke` 调用 Rust command。
- 入口文件：`src/services/tauriApi.ts`
- command 注册：`src-tauri/src/lib.rs` 的 `invoke_handler`。

## 2. 命名约定

- 前端 TypeScript 以 `camelCase` 为主。
- Rust 请求体部分字段以 `snake_case`（如 `class_id`, `student_number`）。
- Rust 返回体通过 `serde(rename = ...)` 将字段序列化为前端期望的 camelCase（如 `createdAt`, `studentCount`）。

## 3. Class API

### 3.1 `get_classes`

- 调用：`classApi.getAll()`
- 参数：无
- 返回：`Class[]`

### 3.2 `create_class`

- 调用：`classApi.create({ name, description? })`
- 请求：
  - `request.name: string`
  - `request.description?: string`
- 返回：`Class`

### 3.3 `update_class`

- 调用：`classApi.update(id, partial)`
- 请求：
  - `id: string`
  - `request.name?: string`
  - `request.description?: string`
- 返回：`Class`

### 3.4 `delete_class`

- 调用：`classApi.delete(id)`
- 请求：`id: string`
- 返回：`void`

## 4. Student API

### 4.1 `get_students`

- 调用：`studentApi.getAll()`
- 参数：无
- 返回：`Student[]`

### 4.2 `get_students_by_class`

- 调用：`studentApi.getByClass(classId)`
- 请求：`classId: string`
- 返回：`Student[]`

### 4.3 `create_student`

- 调用：`studentApi.create({ name, studentNumber, points, classId, className })`
- 请求（传给后端）：
  - `request.name`
  - `request.student_number`
  - `request.points`
  - `request.class_id`
- 返回：`Student`

### 4.4 `update_student`

- 调用：`studentApi.update(id, partial)`
- 请求：
  - `id: string`
  - `request.name?`
  - `request.points?`
  - `request.class_id?`
- 返回：`Student`

> 注意：当前前端 `studentApi.update` 未透传 `student_number`，与后端能力不完全一致。

### 4.5 `delete_student`

- 调用：`studentApi.delete(id)`
- 请求：`id: string`
- 返回：`void`

## 5. Product API

### 5.1 `get_products_by_class`

- 调用：`productApi.getByClass(classId)`
- 请求：`classId: string`
- 返回：`Product[]`

### 5.2 `create_product`

- 调用：`productApi.create({ name, points, stock, classId })`
- 请求：
  - `request.name`
  - `request.points`
  - `request.stock`
  - `request.class_id`
- 返回：`Product`

### 5.3 `update_product`

- 调用：`productApi.update(id, partial)`
- 请求：
  - `id: string`
  - `request.name?`
  - `request.points?`
  - `request.stock?`
- 返回：`Product`

### 5.4 `delete_product`

- 调用：`productApi.delete(id)`
- 请求：`id: string`
- 返回：`void`

## 6. Purchase API

### 6.1 `create_purchase_record`

- 调用：`purchaseApi.create(productId, studentId, quantity)`
- 请求：
  - `request.product_id`
  - `request.student_id`
  - `request.quantity`
- 返回：`PurchaseRecord`
- 可能错误：`库存不足`、`积分不足`

### 6.2 `get_purchase_records_by_class`

- 调用：`purchaseApi.getByClass(classId)`
- 请求：`classId: string`
- 返回：`PurchaseRecord[]`

### 6.3 `get_purchase_records_paginated`

- 调用：`purchaseApi.getByClassPaginated(classId, page, pageSize)`
- 请求：
  - `classId: string`
  - `page: number`
  - `pageSize: number`
- 返回：
  - `records`
  - `total`
  - `totalPages`
  - `currentPage`
  - `pageSize`

### 6.4 `update_shipping_status`

- 调用：`purchaseApi.updateShippingStatus(recordId, status)`
- 请求：
  - `recordId: string`
  - `request.shipping_status: 'pending' | 'shipped'`
- 返回：`void`

## 7. 文件 API

### 7.1 `save_file_to_desktop`

- 调用：`fileApi.saveToDesktop(filename, data)`
- 请求：
  - `filename: string`
  - `data: number[]`（前端 `Uint8Array` 转 `Array<number>`）
- 返回：写入后的本地路径字符串

## 8. 错误处理策略

- Rust command 统一返回 `Result<..., String>`，错误被 `to_string()` 后透传。
- 前端多数调用在页面层 `try/catch` 处理并显示 Toast。
- 对关键状态变更（积分、发货）采用乐观更新 + 失败回滚策略。

## 9. API 演进建议

- 为 `update_student` 前端封装补齐 `student_number` 透传。
- 统一错误码（而不是纯字符串）以支撑前端可控分支处理。
- 对批量导入提供后端批量接口，减少 N 次 command 调用开销。
