# 发布流程说明（Windows）

## 1. CI 文件

- `.github/workflows/release-windows.yml`

## 2. 触发方式

- 推送 tag：`v*`（例如 `v1.0.0`）
- 手动触发：`workflow_dispatch`

## 3. CI 主要步骤

1. Checkout 仓库
2. 安装 Rust stable
3. 安装 Node.js 22
4. 安装 pnpm 9
5. `pnpm install`
6. `pnpm tauri build`
7. 将 `src-tauri/target/release/bundle/nsis/*.exe` 发布到 GitHub Release

## 4. 产物说明

- 当前 workflow 只发布 Windows NSIS 的 `.exe` 安装包。
- 与 `tauri.conf.json` 中 `bundle.targets = "nsis"` 保持一致。

## 5. 维护建议

- 若未来支持 macOS/Linux，请按平台拆分 job 与产物路径。
- 建议补充校验步骤：`pnpm lint`、构建前测试、版本号一致性检查。
- 建议固定 action 版本并定期升级（目前使用 `@v3/@v1`）。
