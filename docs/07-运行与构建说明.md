# 运行与构建说明

## 1. 环境要求

- Node.js（建议 LTS）
- pnpm
- Rust toolchain（用于 Tauri 后端）
- Tauri 2 运行依赖（按操作系统安装）

## 2. 安装依赖

```bash
pnpm install
```

## 3. 前端开发（仅 Web）

```bash
pnpm dev
```

- 默认端口：`1420`
- Vite 别名：`@ -> src`

## 4. 桌面开发（Tauri）

```bash
pnpm tauri dev
```

- `tauri.conf.json` 中配置：
  - 开发 URL：`http://localhost:1420`
  - beforeDevCommand：`pnpm dev`

## 5. 前端构建

```bash
pnpm build
```

- 执行顺序：`tsc && vite build`

## 6. 桌面打包

```bash
pnpm tauri build
```

- 当前配置目标：`nsis`（Windows 安装包）。

## 7. 代码质量命令

```bash
pnpm lint
pnpm lint:fix
```

## 8. 配置文件速览

- `vite.config.ts`
  - Tauri 开发端口固定 1420
  - HMR 端口 1421（当 `TAURI_DEV_HOST` 存在时）
  - 忽略监听 `src-tauri/**`
- `eslint.config.js`
  - 基于 `@antfu/eslint-config`
  - 关闭了若干 React Hooks 规则与按钮 type 规则
- `src-tauri/tauri.conf.json`
  - 应用名：积分豆
  - 主窗口：1200x800

## 9. 数据文件说明

- SQLite 数据库：`pointhub.db`（由后端自动创建）。
- Excel 导入模板：运行时可由页面按钮下载到桌面。

## 10. 调试入口建议

- 前端问题：先看浏览器控制台（Tauri WebView DevTools）。
- 后端问题：看 Rust 命令报错字符串（前端 `catch` 打印）。
- 数据问题：直接检查 SQLite 文件中的 4 张核心表。
