# 商品大转盘冻结需求（MVP）

> 状态：已冻结（2026-03-06）
> 来源：`docs/09-商品大转盘需求澄清.md` 已确认项

## 1. 功能定位

在班级商城中提供“积分大转盘”玩法，老师可为学生执行抽奖。学生用较少积分参与，有机会获得更高价值商品。

## 2. MVP 边界

- 入口：班级商城页新增“积分大转盘”。
- 配置：班级商城设置页新增“转盘配置”Tab。
- 抽奖模式：单抽。
- 奖池：复用现有商品（`products`）。
- 空奖：MVP 不允许空奖。
- 保底：不做。
- 频率限制：不做。
- 结果展示：弹窗。
- 动画：基础版（转盘旋转 + 指针高亮）。

## 3. 关键规则

### 3.1 角色与操作

- 老师代学生操作，不引入权限系统。
- 抽奖前必须先选择学生。

### 3.2 积分规则

- 每班级只有一个转盘配置。
- 每次抽奖消耗积分为班级级配置值。
- 开始前展示：当前积分 / 消耗积分 / 预计剩余积分。
- 若积分不足，按钮禁用并显示红色提示文案。
- 点击开始时，后端再做原子校验。

### 3.3 概率与奖池规则

- 奖品概率通过“重复格子”间接控制（等比）。
- 老师通过可视化转盘添加重复格子（非权重输入框）。
- 每个格子绑定一个商品，抽奖等概率命中任一格子。
- 概率总和按格子自动构成 100%。

### 3.4 库存规则

- K1-A：若奖池中任一奖品库存不足（<=0），整盘禁止转动。

### 3.5 扣分与落库

- 扣分时机：点击“开始抽奖”即扣分。
- 中奖后写入 `purchase_records`。
- 新增来源字段 `source`，值采用中文：`购买` / `抽奖`。

### 3.6 历史记录

- 历史展示采用 K3-A：复用购物记录页。
- 购物记录页新增来源筛选（全部/购买/抽奖）。

## 4. 数据结构变更要求

- `purchase_records` 新增字段：`source`。
- 新增转盘配置表（班级维度）：
  - 抽奖消耗积分
  - 配置更新时间
- 新增转盘格子表（格子列表）：
  - 绑定商品 ID
  - 格子顺序（用于动画落点）

## 5. 后端接口要求

- 获取班级转盘配置。
- 保存班级转盘配置（消耗积分 + 格子商品列表）。
- 执行抽奖（原子事务）：
  - 校验配置与库存
  - 校验学生积分
  - 扣积分
  - 随机命中格子
  - 扣减中奖商品库存
  - 写入购买记录（source=抽奖）

## 6. 前端页面要求

- 商城页：新增“积分大转盘”入口。
- 商城设置页：新增“转盘配置”Tab。
- 转盘页：学生下拉选择、积分展示、禁用态、旋转动画、结果弹窗。
- UI 技术选型：使用开源库 `react-custom-roulette` 渲染转盘，允许通过样式参数统一视觉风格。
- 购物记录页：新增来源筛选与来源展示。
